Logged In User

If your website provides anything to the slot 'user-name' then it will know that the user is logged in. If this is not provided, it will believe no one is logged in.

Components Demo Site Sign In Sign Out UserFirstName 

<byu-header>
  <span slot="site-title">Components Demo Site</span>
  <byu-user-info slot="user">
    <a slot="login" href="#login">Sign In</a>
    <a slot="logout" href="#logout">Sign Out</a>
    <span slot="user-name">UserFirstName</span>
  </byu-user-info>
</byu-header>

Overriding CMS Styles

Many Content Management Systems will add extra divs and spans around your login/logout links. Unfortunately, Web Components only allow you to style directly in the Shadow DOM. To fix this, you'll need some custom CSS. The exact CSS needed will vary depending on your CMS, but this should give you the idea.

Components Demo Site
Sign In
Sign Out 

<style>
  .my-custom-login-class a {
  color: white;
  font-family: 'IBM Plex Sans', Helvetica, Arial, sans-serif;
  cursor: pointer;
  }

  .my-custom-login-class a:hover {
  color: white;
  text-decoration: none;
  }
</style>

<byu-header>
  <span slot="site-title">Components Demo Site</span>
  <byu-user-info slot="user">
    <span slot="login" class="my-custom-login-class">
      <div class="someCMSClass">
        <a class="someOtherCMSClassAndAttributes">Sign In</a>
      </div>
    </span>
    <a slot="logout" href="#logout">Sign Out</a>
  </byu-user-info>
</byu-header>
